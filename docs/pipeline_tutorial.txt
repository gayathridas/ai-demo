How Our Code Gets to the Internet (A 6th Grade Guide)
======================================================

Imagine you love building Lego castles. You build an amazing castle in your bedroom (your computer). But you want your friends across town to see it! You can't just leave it in your room; you need a delivery truck to carefully move it to a museum display window.

In our coding world:
- The Lego Castle = Your website code (index.html, styles.css)
- Your Bedroom = Your Local Computer
- The Delivery Truck = GitHub Actions (The "Pipeline")
- The Museum Window = GitHub Pages (The Internet)

Whenever you finish a new part of your castle, you push a magical button, and a robot helper (GitHub Actions) comes to your room, packs up the castle, and drives it straight to the display window!

Here is what that looks like:

The Robot's Checklist (What happens inside the Pipeline):
---------------------------------------------------------

If you look inside our 'deploy.yml' file, you will see the robot has 4 specific chores:

1. "Checkout": The robot makes a clean copy of your newest code. (Copies the Lego instructions).
2. "Setup Pages": The robot prepares the display shelf at the museum.
3. "Upload Artifact": The robot puts all your website files into one easy-to-carry box. (An artifact is just a fancy word for a packed box).
4. "Deploy to GitHub Pages": The robot opens the box and places your shiny new website on the shelf for the whole world to look at!

And the best part? The robot does all of this completely by itself every single time you click the magic button. That is why it's called an "Automated Pipeline"!


How to Push the Magic Button (Commands for Your Windows Laptop)
---------------------------------------------------------------

To tell the robot to start its work, you need to use a special window on your computer called the "Terminal" or "PowerShell". 

Follow these 3 easy steps:

1. Create a New Home on GitHub:
   Go to GitHub.com and create a new repository. Give it a name like "ai-nexus-demo".

2. Link Your Computer to GitHub:
   Open your Terminal (you can search for "PowerShell" in your Start menu), and type these commands one by one. 

   git remote add origin https://github.com/gayathridas/ai-demo.git

   git branch -M main

3. Send the Code (Push the Magic Button!):
   Now, type this command and press Enter:

   git push -u origin main

That's it! As soon as you press Enter on that last command, the robot helper (GitHub Actions) will wake up and start building your website. You can watch it work by clicking the "Actions" tab on your GitHub page!


🆘 Troubleshooting: "The Robot is Stuck/Error"
---------------------------------------------

If you see an error that says "Get Pages site failed" or "Not Found", don't worry! It just means you haven't told GitHub to listen to the Robot yet. 

Follow these 3 "Final Fix" steps:

1. Click on "Settings" at the top of your GitHub repository.
2. On the left side, click on "Pages".
3. Under "Build and deployment", look for "Source". 
   Change it from "Deploy from a branch" to "GitHub Actions".

Once you do that, go back to the "Actions" tab and click "Re-run all jobs". The robot will now be allowed to place the castle in the museum window!
