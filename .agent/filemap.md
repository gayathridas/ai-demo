# Project File Map

This document serves as an index mapping the overall structure and purpose of the project files to help navigate the repository.

## 📂 Root Directory (`/`)
- **`index.html`**
  - **Purpose:** The main entry point for the frontend, defining the document structure and primary UI layout.
- **`styles.css`**
  - **Purpose:** The central stylesheet. Handles all custom CSS variables, responsive design rules, glassmorphism UI, and dark mode themes.
- **`script.js`**
  - **Purpose:** Client-side interactions, handling click animations and console output for the demo.
- **`agent.md`**
  - **Purpose:** General project context, architecture, frameworks, and deployment guidelines for AI agentic workflows.

## 📂 AI and Workflows (`/.agent/`)
- **`filemap.md`**
  - **Purpose:** This file; an up-to-date registry of file locations and purposes.
- **`coding_standards.md`**
  - **Purpose:** Defines the strict rules, formatting conventions, and SDLC practices followed by the AI.
- **`workflows/feature.md`**
  - **Purpose:** Slash command `/feature` definition for adding UI components.
- **`workflows/pipeline.md`**
  - **Purpose:** Slash command `/pipeline` definition for generating the GitHub Actions CI/CD deployment pipeline.

## 📂 Documentation (`/docs/`)
- **`ascii_flow.md`**
  - **Purpose:** ASCII diagrams illustrating the frontend architecture, planned deployment flow, and AI agent context rules.
- **`pricing_tab_plan.md`**
  - **Purpose:** Implementation plan for adding the Pricing tab to the UI.
- **`pricing_ascii.md`**
  - **Purpose:** ASCII visualization of the header changes for the Pricing tab feature.
- **`pipeline_tutorial.txt`**
  - **Purpose:** A simple, 6th-grade level explanation of how the GitHub Actions pipeline works, including an ASCII flow.
